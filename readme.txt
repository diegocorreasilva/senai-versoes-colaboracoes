principais comandos utilizados para gerenciar o versionamento mostrados na UC

git config --global user.name "seu nome" = todos commits alterado por voce tera seu nome

git config --global user.email " seu email" =todos commits alterado por voce tera seu email

git init = iniciou projeto no repositório local

git add = Adiciona arquivos ao histórico do projeto, na staging.

git status = Verifica o status das alterações realizadas no repositório.

git commit -m = salvar o arquivo no repositório

git log = visualizar as alterações feitas

git remote add origin = Informa a pasta remota.

git remote -v = visualizar o repositório remoto

git pull =  atualiza o repositório local com alteraçoes feita no repositorio remoto

git push u- origin master = Publica as alterações no repositório remoto.

git checkout master " main" = retonar para o ramo principal

git checkout -b "nome da branch" = criar uma nova branch

git merge "nome da a ser unificada" = unifica duas ou mais branch

git tag -a = cria uma tag

git tag = visualiza a tag criada
